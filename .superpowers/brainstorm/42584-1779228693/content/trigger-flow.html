How should the auto-trigger interrupt a command?

When a user runs oracle generate "..." for the first time in a project

A

Silent takeover

Onboarding starts immediately, no prompt

terminal
$ oracle generate "test the login flow"

⚡ Oracle Setup — first time in this project

Choose a provider:
❯ claude openai gemini

ANTHROPIC_API_KEY: █
B

Ask first

Pauses and asks permission before starting onboarding

terminal
$ oracle generate "test the login flow"

! Oracle isn't configured for this project yet.
Run setup now? [Y/n] █

(skip with --no-setup or run later: oracle setup)
C

Warn and fail gracefully

Doesn't run onboarding inline — tells user what to do

terminal
$ oracle generate "test the login flow"

✗ Oracle isn't configured for this project.
Run oracle setup to get started.

$ oracle setup
